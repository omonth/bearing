# PRD: 顾客端商城生产就绪

**标签**: `ready-for-agent`

## 问题陈述

轴承销售系统后端架构已完全重构并通过全部测试，但顾客端存在以下问题使之无法对真实客户开放：

1. **无 HTTPS/域名**：客户只能通过 `http://IP` 访问，无法信任、无法搜索引擎收录
2. **前端只有中文硬编码**：非中文客户无法使用
3. **产品图片为占位符**：所有轴承显示 `via.placeholder.com` 灰色方块
4. **支付仅沙箱模式**：支付宝/微信/银联全部模拟，无法真实收款
5. **无客户自注册**：客户只能由管理员在后台创建
6. **无响应式设计验证**：移动端体验未经测试

## 解决方案

分两个阶段使商城生产就绪：

**第一阶段——立即可上线（本 PRD 范围）：**
- Nginx 配置 HTTPS（Let's Encrypt 自动续期）
- 产品图片上传与管理（已有 API，缺实际图片资源）
- 前端国际化基础（中/英双语切换）
- 移动端响应式适配
- SEO 基础（meta 标签、sitemap）

**第二阶段——后续 PRD：**
- 真实支付对接（需企业资质）
- 客户自注册/登录
- 邮件通知（订单确认、发货通知）
- 访问统计与分析

## 用户故事

1. 作为客户，我想通过 `https://我的域名.com` 访问商城，以便我信任这个网站
2. 作为客户，我想在手机上顺畅浏览和购买轴承，以便我在车间现场就能下单
3. 作为客户，我想看到真实的轴承产品图片，以便我能确认产品外观
4. 作为国际客户，我想切换到英文界面，以便我能理解产品信息
5. 作为搜索引擎爬虫，我想抓取产品页面和 sitemap，以便轴承商城能被潜在客户搜索到
6. 作为管理员，我想上传产品图片并自动优化，以便商城看起来专业

## 实现决策

### HTTPS 配置

- Let's Encrypt 免费证书，certbot + Nginx 自动续期
- Docker Compose 中添加 certbot 服务，定时检查续期
- HTTP 自动重定向到 HTTPS
- 证书存储在 Docker volume 中持久化

### 前端国际化

- 使用 Next.js 内置 i18n 路由（Pages Router 支持）
- 语言文件：`locales/zh.json`、`locales/en.json`
- 语言切换按钮在 Header 中
- 默认中文，支持 `?lang=en` 参数和浏览器语言检测
- 仅翻译 UI 文本，产品数据保持中文（轴承行业术语以中文为主）

### 响应式设计

- 现有 CSS 已为桌面端设计，需加媒体查询适配移动端
- 产品网格：桌面 4 列 → 平板 2 列 → 手机 1 列
- 购物车：桌面侧滑面板 → 手机全屏抽屉
- Header：桌面完整导航 → 手机汉堡菜单
- 图片懒加载优化移动端性能

### 产品图片

- 管理员通过 admin.html 上传产品图片（已有 API）
- 前端 `next.config.js` 已配置图片代理到后端
- 添加图片压缩/缩放（sharp）减少加载时间
- 默认占位图改为本地 SVG（不依赖外部 CDN）

## 测试决策

- HTTPS：手动验证证书有效性 + 浏览器锁图标
- 响应式：Chrome DevTools 模拟 iPhone/Android + 实际手机测试
- 国际化：手动切换语言验证所有 UI 文本
- 图片上传：通过 admin.html 上传并验证前端显示
- 向后兼容：27 个后端测试保持全通过

## 超出范围

- 真实支付对接（需企业营业执照、商户号）
- 客户自注册/登录系统
- 邮件服务（订单通知）
- 性能优化（CDN、缓存策略）
- 管理后台独立前端

## 附加说明

- 域名需要用户自行购买（约 60 元/年），本 PRD 仅配置 Nginx + certbot
- Let's Encrypt 证书需要在域名 DNS 解析生效后才能申请
- 国际化先做中英双语，后续可按需扩展
